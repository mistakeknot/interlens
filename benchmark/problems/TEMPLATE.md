# Test Problem Template

## Problem ID
`domain_problem-name`

## Domain
[code | design | strategy | product | team]

## Title
Brief problem title

## Context
Background information about the situation:
- Organization/team context
- Current state
- Relevant history
- Stakeholders

## Challenge
The specific problem or decision to be made:
- What's not working?
- What needs to be decided?
- What tension exists?

## Constraints
Limitations and boundaries:
- Technical constraints
- Resource constraints (time, budget, people)
- Organizational constraints
- Must-haves vs nice-to-haves

## Success Criteria
What does a good solution look like?
- Specific outcomes
- Measurable improvements
- Stakeholder satisfaction
- Implementation feasibility

## Prompt for Agent
The actual prompt given to the agent:

```
[Context paragraph]

[Challenge statement]

[Optional constraints]

Your role: [What the agent should do]
```

## Expected Lens Relevance
Which lenses would be particularly relevant? (for evaluation scoring)

- High relevance: [Lens names that directly apply]
- Medium relevance: [Lenses that could apply with adaptation]
- Low relevance: [Lenses that are a stretch]

## Baseline Solution Pattern
What would a typical solution look like WITHOUT linsenkasten?

- Common approaches
- Typical recommendations
- Estimated originality: [2-4/10]
- Estimated novelty: [2-4/10]

## Target Solution Pattern
What could a solution look like WITH linsenkasten?

- Lens-based reframes
- Novel connections

Additional context for evaluators:

- What makes this problem interesting?
- What creative leaps are possible?
- What would surprise you?
